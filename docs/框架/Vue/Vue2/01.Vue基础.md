## 1、模板语法 数据绑定

### 1.3. 模板语法

Vue模板语法包括两大类

**1.插值语法**

功能：用于解析标签体内容

写法：

```html
<h1>{{xxx}}</h1>
```

{{}}里写 js 表达式，遇到变量就访问模型数据

**2.指令语法** 

功能：用于解析标签（包括：标签属性、标签体内容、绑定事件…）

举例：`<a v-bind:href="xxx">`或简写为`<a :href="xxx">`，xxx 同样要写 js 表达式，**可以直接读取到 data 中的所有属性**

备注：Vue中有很多的指令，且形式都是 v-xxx，此处只是拿v-bind举例

```html
<body>

    <h2>插值语法</h2>
    <h4>你好，{{ name }}</h4>

    <a v-bind:href="tencent.url.toUpperCase()" x="hello">点我去看{{ tencent.name }}1</a>
```

### 1.4. 数据绑定

Vue中有2种数据绑定的方式

- 单向绑定：**v-bind**
  - **数据只能从 data 流向页面**
  - `v-bind:attr`可简写成`:attr`
- 双向绑定：**v-model**
  - **数据不仅能从 data 流向页面，还可以从页面流向 data**


**备注** 

**双向绑定一般都应用在表单类元素上**，如 `<input> <select> <textarea>`等

`v-model:value`可以简写为`v-model`，因为v-model默认收集的就是value值

```html

  <div id="root">
    <!-- 普通写法 -->
    <!-- 单向数据绑定：<input type="text" v-bind:value="name"><br/> -->
    <!-- 双向数据绑定：<input type="text" v-model:value="name"><br/> -->
    <!-- 简写 -->
    单向数据绑定：<input type="text" :value="name"><br/>
    双向数据绑定：<input type="text" v-model="name"><br/>
    <!-- 如下代码是错误的，因为 v-model 只能应用在表单类元素（输入类元素）上 -->
    <!-- <h2 v-model:x="name">你好啊</h2> -->
  </div> 
</body>

    el: '#root',

      name: 'cess'
    }
  })
</script>
```

## 2、el与data的两种写法

### 1.5. el 与 data 的两种写法

el有2种写法

- 创建Vue实例对象的时候配置el属性

- 先创建Vue实例，随后再通过`vm.$mount('#root')`指定el的值

```js
//第一种
el:'#root',
//第二种
v.$mount('#root'), 
```


data有2种写法

- 对象式：`data： { }`

- 函数式：`data() { return { } }`

```js
//第一种
data: {
	key:'value'
}
//第二种
data() {
	console.log('@@@', this) // 此处的this是Vue实例对象
	return {
		key: 'value'
	}
}
```


如何选择：目前哪种写法都可以，以后到**组件时，data必须使用函数**，否则会报错

一个重要的原则由管理的函数，**一定不要写箭头函数**，**否则 this 就不再是Vue实例了**



## 3、事件处理

### 1.8. 事件处理

#### 1. 事件的基本用法

- 使用`v-on:xxx`或`@xxx`绑定事件，其中 xxx 是事件名
- 事件的回调需要配置在`methods`对象中，最终会在`vm`上
- methods中配置的函数，**不要用箭头函数**，否则 this 就不是vm了
- methods中配置的函数，都是被 Vue所管理的函数，**this 的指向是vm或组件实例对象**
  - **vue构造函数种的this===实例化对象vm===methods中的this**

- `@click="demo"`和`@click="demo($event)"`效果一致，但后者可以传参
  - 函数名不推荐写小括号，只有传递参数时才可以写


```html
<body>
    <div id="app">
        <button @click="showMsg">点击弹出文字</button>
        <button v-on:click="showMsg">点击弹出文字</button>
 	</div>
</body>
```

```js
const vm=new Vue({      
    //声明监控范围
    el:'#app', 
    //创建m模型数据
    data:{
        msg:'张三'
    },
    methods:{
        showMsg(){
            //获取模型数据，this.key
            alert(this.msg)
        }
    }
})
```

- `@click=" "`中`" "`中间也可以直接写js逻辑代码

```html
<button @click="num++">{{num}}</button>
//...
<script>
    data:{
		num:10
	}
</script>
```



#### 2. 事件修饰符

 Vue中的事件修饰符

- **prevent**	**阻止默认事件**（常用）    鼠标右键菜单，链接跳转等

- **stop**		   **阻止事件冒泡**（常用）

- **once**		  **事件只触发一次**（常用）

- capture	  使用事件的捕获模式

- self		      只有`event.target`是当前操作的元素时才触发事件

- passive	    事件的默认行为立即执行，无需等待事件回调执行完毕

- 修饰符可以连续写，比如可以这么用：`@click.prevent.stop="showInfo"`

使用：`@事件类型.修饰符=事件函数`


## 4、计算属性 侦听属性

### 1.9. 计算属性

#### 1.computed计算属性

**定义：**要用的属性不存在，需要通过已有属性计算得来

**原理：**底层借助了**Objcet.defineproperty()**方法提供的**getter**和**setter**

**优势：**与methods实现相比，内部有缓存机制（复用），**效率更高，调试方便**



**备注**：

- 计算属性最终会出现在**vm**上，直接读取使用即可

- 如果计算属性要被修改，那必须写set函数去响应修改，且set中要引起计算时依赖的数据发生改变

- 如果计算属性确定不考虑修改，可以使用计算属性的简写形式



##### **语法：**

```js
new Vue({
    el,
    data,
    methods: { 
    },
    // 声明计算属性
    // 和普通函数区别: 1-调用角度 不加小括号 {{ 函数名Cptd }}、2- 缓存
    computed: {
        函数名Cp() {  
            return 值
        }
    }
})
```

##### 调用：

- `{{  计算属性函数名 }}` 切记切记切记函数升级版 所以**不要加小括号**



#### 2.计算属性缓存清除机制

明确：计算属性缓存了，**后期再调用  代码就不重新执行了，而是直接返回结果**

原因：**提升性能**



思考：缓存了之后  一定是不会再触发吗，数据变化了怎么办？  缓存机制是怎么的呢

回答：底层是【响应式依赖缓存】

也就是：只要计算属性中写的【任意一个】模型数据改变了就会重新触发/重新缓存



#### 3.set和get语法

**语法:**

```js
computed:{
	FnNameCp:{	//计算属性名
		get(){
			return xxx
		},
		set(data){
			//...
		},
                
	},
},
```

**get函数什么时候执行？**

- **初次读取**时会执行一次
- 当**依赖的数据发生改变**时会被再次调用
  - 依赖的数据一般就是return中的数据

- get()就相当于计算属性简写的函数

**set什么时候执行**

- 当计算属性被篡改时，set就会被触发
- set(data),**data是计算属性改变后的新值**
  - 计算属性名=数据    这个数据会变成set的第一个形参
  - 即`<button @click="fullNameCp='赵 四'">赵四</button>`

- 好处：减少视图操作



#### 4.计算属性应用场景

```

vuex大量使用
```

### 1.10. 侦听属性

#### 1. 侦听属性基本用法

**watch**监视属性

1. 当**被监视的属性变化**时，回调函数**自动调用**，进行相关操作

1. 监视的属性必须存在，才能进行监视，**既可以监视data，也可以监视计算属性**

1. 配置项属性**immediate:false**，默认为false,改为 **true**，则**初始化时调用一次** `handler(newValue,oldValue)`函数

1. 监视有**两种写法**

   - 创建Vue时传入`watch: {}`配置项

   - 通过`vm.$watch()`监视

语法：

#### 2. 侦听属性简写

- 需要监听的属性写在**watch配置项**里

- 切记切记**方法名必须是模型数据**（也就是**data中的键**）  => 目的：**写谁监控谁**
- 方法有两个形参
  - 第一个是**改变后的新数据**
  - 第二个是**改变前的数据**
- 视图不能调用，只能重置模型，例如搜索拿到数据后保存到模型中 视图展示

```js
watch: {
	data中的键(新数据，旧数据) { // 形参是系统传递进来的
        ....    
	}
}
```

#### 3. 侦听属性详写

监听属性以**对象**的形式详写

里面包含几个属性：

- **immediate: true**,  初始化时让handler调用一下

- **handler**是**处理监听变动**时的函数，有两个参数，相当于简写

  - 第一个是**改变后的新数据**
  - 第二个是**改变前的数据**

- **deep:true**  深度监听

  

```js
const vm = new Vue({
    data:{
		attr:value
    }
	//...............................省略其他配置项
    // 方式一
    watch:{		
        data中的键:{	
            immediate:true,		//初始化调用handler函数
            handler(newValue,oldValue){ //newValue为改变后的新数据，oldValue为改变前的数据
                console.log('attr被修改了',newValue,oldValue)
            }
        }
    } 
    
})


// 方式二
vm.$watch('attr', {		
	immediate: true, // 初始化时让handler调用一下
	//handler什么时候调用？当isHot发生改变时
	handler(newValue, oldValue) {
		console.log('attr被修改了', newValue, oldValue)
	}
})
```

#### 4. 深度侦听

1. Vue中的**watch**默认不监测对象内部值的改变（一层）

1. 在**watch**中配置`deep:true`**可以监测对象内部值的改变**（多层）

注意

1. Vue自身可以监测对象内部值的改变，但Vue提供的watch默认不可以

1. 使用watch时根据监视数据的具体结构，决定是否采用深度监视

```js
<title>天气案例_深度监视</title>
<script type="text/javascript" src="../js/vue.js"></script>

<div id="root">
  <h3>a的值是:{{ numbers.a }}</h3>
```

#### 6. 计算属性 VS 侦听属性

**computed**和**watch**之间的区别

- computed能完成的功能，watch都可以完成

- watch能完成的功能，computed不一定能完成，例如watch可以进行异步操作

- 计算属性：计算属性有缓存、并且是响应式依赖缓存，调用不加小括号

- 侦听器：侦听器无缓存，侦听模型数据变化，不能调用


两个重要的小原则

- 所有被Vue管理的函数，最好写成普通函数，这样 this 的指向才是vm或组件实例对象

- 所有不被Vue所管理的函数（定时器的回调函数、ajax 的回调函数等、Promise 的回调函数），最好写成箭头函数，这样 this 的指向才是vm或组件实例对象

  

使用计算属性

```vue
<div>{{fullName}}</div>
...
new Vue({
    el:'#root', 
    data:{ 
        firstName:'张',
        lastName:'三'
    },
    computed:{
    	fullName(){
		    return this.firstName + '-' + this.lastName
    	}
    }
})
```

使用监听属性

```js
new Vue({
  el:'#root',
  data:{
    firstName:'张',
    lastName:'三',
    fullName:'张-三'
  },
  watch:{
    firstName(val){
      setTimeout(()=>{
        this.fullName = val + '-' + this.lastName
      },1000);
    },
    lastName(val){
      this.fullName = this.firstName + '-' + val
    }
  }
})
```

### 1.13. 列表渲染

#### 1. 基本列表

- 可遍历：**数组、对象、字符串、数字**（用的少）、指定次数（用的少）

#### 2. key的作用与原理

原理



面试题：react vue中的**key**有什么作用？（key的内部原理）

1.虚拟DOM中key的作用：**key是虚拟DOM中对象的标识**，当数据发生变化时，**Vue会根据新数据生成新的虚拟DOM，随后Vue进行新虚拟DOM与旧虚拟DOM的差异比较**，比较规则如下

2.对比规则

- 旧虚拟DOM中找到了与新虚拟DOM相同的key

  - 若虚拟DOM中内容没变, 直接使用之前的真实DOM

  - 若虚拟DOM中内容变了, 则生成新的真实DOM，随后替换掉页面中之前的真实DOM

- 旧虚拟DOM中未找到与新虚拟DOM相同的key创建新的真实DOM，随后渲染到到页面

3.用**index**作为**key**可能会**引发的问题**

- 若对数据进行**逆序添加**、**逆序删除等破坏顺序操作**，会产生没有必要的真实DOM更新 ==> 界面效果没问题，但效率低

- 若结构中还包含**输入类的DOM**：会产生错误DOM更新 ==> 界面有问题

开发中如何选择key？

**最好使用每条数据的唯一标识作为key，比如 id、手机号、身份证号、学号等唯一值**

如果不存在对数据的逆序添加、逆序删除等破坏顺序的操作，仅用于渲染列表，使用index作为key是没有问题的

### 1.14. 收集表单数据

#### v-model

v-model专门用来双向绑定**表单**中的数据

- 若`<input type="text"/>`，则v-model收集的是value值，用户输入的内容就是value值

- 若`<input type="radio"/>`，则v-model收集的是value值，且要给标签配置value属性

### 1.15. 过滤器

定义：对要显示的数据进行特定格式化后再显示（适用于一些简单逻辑的处理）

#### 注册过滤器：

1.全局过滤器

```js
Vue.filter(name, (管道左边的数据，小括号里面的数据)=>{
	return 结果

```

2.局部过滤器

`new Vue {filters: {}}` 

#### 使用过滤器：

```
{{ data中的键 | 过滤器名(实参2,...)}} 
或 
v-bind:属性 = "xxx | 过滤器名(实参2,...)"
```

备注：

- 过滤器可以接收额外参数，多个过滤器也可以串联

- 并没有改变原本的数据，而是产生新的对应的数据

处理时间的库moment体积较大 dayjs轻量级

```vue
		<div id="root">
			<h2>时间</h2>
            <h3>当前时间戳：{{time}}</h3>
            <h3>转换后时间：{{time | timeFormater()}}</h3>
			<h3>转换后时间：{{time | timeFormater('YYYY-MM-DD HH:mm:ss')}}</h3>

                time:1626750147900,
            },
```

### 1.17. 自定义指令

**directives**配置项

#### 定义语法

##### 1.局部指令

```js
new Vue({															
  directives:{ 
    指令名:配置对象 
  }   
})

new Vue({															
  directives:{ 
      	bind: function () {},				//显示，挂载
  		inserted: function () {},			//显示，挂载   可以获取父节点
  		update: function () {},				//更新
  		unbind: function () {}				//隐藏
  }   
})
```

##### 2.全局指令

```js
Vue.directive('fbind', {
    // el就是DOM元素，binding就是要绑定的对象
    bind(el, binding) {},
    inserted(el, binding) {},
    update(el, binding) {},
    unbind(el, binding) {}
})
```



配置对象中常用的3个**回调函数**

| 钩子函数                            | 作用                                                         |
| ----------------------------------- | ------------------------------------------------------------ |
| **bind(element, binding)**          | 只调用一次，指令第一次绑定到元素时调用。在这里可以进行一次性的初始化设置。 |
| **inserted(element,bindingparmas)** | 被绑定元素插入父节点时调用 (仅保证父节点存在，但不一定已被插入文档中)。 |
| **update(element, binding)**        | 指令所在模板结构被重新解析时调用                             |

- **el**：指令所绑定的元素，可以用来直接操作 DOM。
- **binding**：一个对象，包含以下 property：
  - `name`：指令名，不包括 `v-` 前缀。
  - `value`：指令的绑定值，例如：`v-my-directive="1 + 1"` 中，绑定值为 `2`。
  - `oldValue`：指令绑定的前一个值，仅在 `update` 和 `componentUpdated` 钩子中可用。无论值是否改变都可用。
  - `expression`：字符串形式的指令表达式。例如 `v-my-directive="1 + 1"` 中，表达式为 `"1 + 1"`。
  - `arg`：传给指令的参数，可选。例如 `v-my-directive:foo` 中，参数为 `"foo"`。
  - `modifiers`：一个包含修饰符的对象。例如：`v-my-directive.foo.bar` 中，修饰符对象为 `{ foo: true, bar: true }`。
- `vnode`：Vue 编译生成的虚拟节点。移步 [VNode API](https://v2.cn.vuejs.org/v2/api/#VNode-接口) 来了解更多详情。
- `oldVnode`：上一个虚拟节点，仅在 `update` 和 `componentUpdated` 钩子中可用



##### 函数简写

在很多时候，你可能想在 `bind` 和 `update` 时触发相同行为，而不关心其它的钩子。比如这样写：

```js
Vue.directive('color-swatch', function (el, binding) {
  el.style.backgroundColor = binding.value
})
```

#### 3. 总结生命周期

**总结**

常用的生命周期钩子

```
created   异步请求
mounted   异步请求、DOM操作（swiper、echarts、聊天默认滚到底部等等）
updated   监控数据变化进一步DOM操作，例如聊天窗口到底部、订单可视化图表重置等等，切记数据必须页面显示才能监控
destroyed 清理非vue资源防止内存泄露，例如登陆倒计时定时器
```

关于销毁Vue实例

1. 销毁后借助Vue开发者工具看不到任何信息

1. 销毁后自定义事件会失效，但原生DOM事件依然有效

1. 一般不会在**beforeDestroy**操作数据，因为即便操作数据，也不会再触发更新流程了

```vue
<title>引出生命周期</title>
<script type="text/javascript" src="../js/vue.js"></script>

<div id="root">
  <h2 :style="{opacity}">欢迎学习Vue</h2>
  <button @click="opacity = 1">透明度设置为1</button>
  <button @click="stop">点我停止变换</button>
</div>

<script type="text/javascript">
  Vue.config.productionTip = false
  new Vue({
    el: '#root',
    data: {
      opacity: 1
    },
    methods: {
      stop() {
        this.$destroy()
      }
    },
    // Vue完成模板的解析并把初始的真实DOM元素放入页面后（挂载完毕）调用mounted
    mounted() {
      console.log('mounted', this)
      this.timer = setInterval(() => {
        console.log('setInterval')
        this.opacity -= 0.01
        if (this.opacity <= 0) this.opacity = 1
      }, 16)
    },
    beforeDestroy() {
      clearInterval(this.timer)
      console.log('vm即将驾鹤西游了')
    },
  })
</script>
```
