# 五、TypeScript

一个不错的Ts学习网站：TypeScript 入门教程 (opens new window)

# 1.1 是什么

说明：TypeScript是JavaScript的一个超集，主要提供了 类型系统 和对 ES6 的支持，它由Microsoft微软开发。

组成：JavaScript + type 类型系统 + （some other stuff）一些其他东西例如面向对象一些内容

留心：项目使用ts只是为了更好的开发，最终打包上线还得js代码，浏览器不支持解析ts文件

# 1.2 为啥学

1、调用函数限制参数类型，例如vue调用组件props的价值

2、编译出现错误提示，减少运行错误（不需要等待代码运行后发现分析逻辑BUG或）

3、代替jsdoc (opens new window)

4、有代码提示

等等

增加了静态类型，可以在开发人员编写脚本时检测错误，使得代码质量更好，更健壮。
优势:
1. 杜绝手误导致的变量名写错;
2. 类型可以一定程度上充当文档;
3. IDE自动填充，自动联想;

# 1.3 编译并运行

1.下载ts模块

cnpm i typescript -g    或   yarn global add typescript

2.生成tsconfig.json文件，在项目目录下

tsc --init

3.要想使用ts最新的特性就得修改tsconfig.json文件的“target”为最新标准es6

{
    "compilerOptions": {
        // ....
        "target": "es6",  
        // ....
    }
}

4.安装vscode插件code Runner，在ts文件内右键Run Code即可运行

# 1.5 接口interface

interface接口 :命名数据结构（例如对象）的另一种方式；

与type 不同，interface仅限于描述对象类型。

java角度：定义个接口类，后期其他类必须按照这个标准来实现

# 1.5.1 语法

接口用interface来定义

interface 接口名称{
	键: 数据类型
}

# 标准普通对象

interface 接口名称 {
	 name: string,
	 sex: string | number,
	 age?: number,
	 readonly desc: string
}
// 使用接口类声明对象
let obj:名称 = {
    name:'hello', 
    sex: '男',
    desc: 'demo'
}

# 标准特殊对象Array

interface Iobj2 {
    id:number,
    content:string
}
interface Iarr {
    // 索引名: 索引类型: 值的类型
    // [i: number]: number | string | {id:number,content:string}
    [i: number]: number | string | Iobj2
}
const arr1: Iarr = [11,22,33]
const arr2: Iarr = [11,'12312',33]
const arr3: Iarr = [11,'12312',{ id:1,content:'内容' }]

(形参:类型,...,):返回值的类型

// 使用接口类定义函数
let 函数名:fun = function(num1,num2) {
	return num1+num2
}

# 1.5.3 接口类型和别名区别

相同点：都可以给对象指定类型

不同点

  • 接口-重名合并，别名-报错

    a:string,

interface A {
    b:number
}
  • 接口-只能描述对象，别名-既可以描述对象又可以描述任意类型
type strAndNum = string | number

等等

# 1.6.4 修饰符

public 默认修饰符，TypeScript中类中的成员默认为public

private 类的成员不能在类的外部访问，子类也不可以

protected 类的成员不能在类的外部访问，但是子类中可以访问。如果一个类的构造函数，修饰符为protected，那么此类只能被继承，无法实例化。

readonly 关键字readonly可以将实例的属性，设置为只读

					
	public  	 自身	子类	外部
	protected   自身	子类 
	private     自身


class Animal
{
    public a = 1
    protected b = 2
    private c = 3
}

class Dog extends Animal {

    readonly d = 4

    }
}

const dog1 = new Dog

console.log(3, dog1.d)

概念：泛型就是把类型当成参数

# 1.12.1 基本使用

注意：<>写在()前

  return arg;

test<number>(111);// 返回值是number类型的 111

# 1.12.2 索引签名

索引签名的思想是在只知道键和值类型的情况下对结构未知的对象进行类型划分。

用处：当你有一个对象 里面有很多键 ，一个个写比较麻烦，写核心的，其他带过

# 对象

interface Obj {
     // 对象的键的类型都是string
	 [key: string]: number
}

const obj: Obj = {a:1,b:2}

使用

interface Iobj {
    a: number,
    b: number,
    // 描述对象剩余的键都是字符串
    // 描述对象剩余的类型都是任意类型
    [key: string]: any
}

const obj: Iobj = {
    // 键和值
    // 键值对
    // 索引和值
    a: 1,
    b: 2,
    c: 'sadda',
    d: 'fasdfsdf',
    // ..
}

# 数组

interface Iarr {
    // 数组的键是下表，所以类型是number
    [index:number]: number
}
const arr: Iarr = [111,222,33]


// 使用泛型
interface Arr<T> {
	 [insert: number]: T
}
const arr: Arr<string> = ['a', 'b']
